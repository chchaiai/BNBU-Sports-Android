## 周目标是什么

本周目标是完成 BNBU Sports Android 端第一版本的环境熟悉、核心功能测试、问题修复和阶段性汇报材料整理。

3. 检查腾讯云服务器、宝塔面板、数据库连接和 LightCOS 存储桶情况。
4. 对照 14 个核心功能完成 Android 端主流程测试。
5. 重点验证打卡签到、图片/视频上传、成绩展示、耐力跑换算、免测申请等学生端功能。
6. 发现并修复 Android 端可处理的问题。
7. 整理测试进度表、问题清单和周报，为周会汇报做准备。

## 遇到哪些困难，踩到了哪些坑

1. LightCOS 后端真实入桶仍需确认。
   - 腾讯云上已经能看到项目存储桶。
   - Android 端图片/视频上传和提交审核流程已跑通。
   - 但此前接口返回表现为 `/uploads/...`，更像服务器本地路径，暂时不能完全证明文件已由后端上传到 LightCOS。

2. 耐力跑换算后端规则存在异常。
   - 手测发现男生 1000m 输入较慢时间时曾返回 100 分。
   - API 检查也发现慢速时间存在异常高分返回。
   - Android 端已加输入范围保护，但最终换算规则需要后端确认。

3. 成绩归档当前无法在 Android 端完整验证。
   - 当前 Android 源码中没有发现成绩归档页面、模型或接口封装。
   - 常见学生端归档接口路径返回 404。
   - 需要确认该模块是否属于本周 Android 学生端范围。

4. 数据导入导出更像 Web / 教师 / 管理员端能力。
   - Android 学生端没有直接导入导出入口。
   - 部分 teacher export 接口可用，但部分 import / admin 相关接口返回 404。
   - 需要确认 Android 端是否需要承担该模块。

5. 拍照上传在模拟器中受环境限制。
   - 选择照片/视频上传已跑通。
   - 拍照按钮原本存在临时文件映射问题，已修复。
   - 当前模拟器无可用相机，只能验证 fallback 提示；真机拍照仍需后续补测。

6. 部分后端字段和 Android 显示字段不一致。
   - 成绩字段存在 `checkin` / `overallTotal` 与 Android 原预期字段不一致的问题。
   - 部分数值字段以字符串形式返回。
   - 通知、记录中存在 `course` 等英文枚举值直接出现在用户可见文案中的情况。
   - Android 已做部分兼容，但建议后端统一字段规范。

7. 源码中仍存在部分中文乱码文案。
   - 当前主流程可用，但部分页面、注释或提示语仍可能出现乱码。
   - 后续需要统一修复编码和文案。

## 是否需要和其他端人员合作协调

需要，主要有以下协作事项：

1. 需要后端负责人确认 LightCOS 上传链路。
   - `/api/upload/proof` 当前是否已经上传到 LightCOS。
   - 当前返回的 `/uploads/...` 是临时本地路径、COS key，还是最终约定格式。
   - 如果尚未接入 LightCOS，需要确认最终接口和接入时间。

2. 需要陈昊确认字段和规则。
   - 成绩接口最终字段规范。
   - 耐力跑换算规则和异常慢速输入边界。
   - 后端枚举字段是否统一，例如 `course` 是否应转换成中文业务文案。

3. 需要产品或负责人确认模块范围。
   - 成绩归档是否属于 Android 学生端本周范围。
   - 数据导入导出是否属于 Android 学生端范围，还是仅属于 Web / 教师 / 管理员端。
   - 当前 14 个核心功能是否可以按“Android 学生端主流程已跑通 + 后端/API待确认项”进行阶段性验收。

4. 需要教师端 / 管理员端配合审核闭环。
   - Android 端已能提交待审核记录。
   - 如果需要验证完整闭环，需要教师端或管理员端对提交记录进行审核。

5. 需要真实 Android 设备补测。
   - 模拟器无法完整验证拍照能力。
   - 后续建议用实体 Android 手机补测拍照上传。
