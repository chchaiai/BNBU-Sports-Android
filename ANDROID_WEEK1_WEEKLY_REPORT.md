## 遇到的问题 / 困难 / 踩到的坑

1. LightCOS 后端真实入桶仍需确认。
   - 腾讯云上已能看到项目存储桶。
   - Android 端图片/视频选择与提交流程已跑通。
   - 但此前 API 返回路径表现为 `/uploads/...`，更像服务器本地文件路径，还不能完全证明文件已经由后端上传到 LightCOS。

2. 耐力跑换算后端规则存在异常。
   - 手测发现男生 1000m 输入 13 分钟曾返回 100 分。
   - 进一步 API 检查也发现慢速时间存在异常高分返回。
   - Android 端已加输入范围保护，但最终换算规则仍需后端确认。

3. 成绩归档功能当前无法在 Android 端完整验证。
   - 当前 Android 源码中没有发现成绩归档页面、模型或接口封装。
   - 常见学生端归档接口路径返回 404。
   - 需要确认本周是否要求 Android 学生端实现该功能，还是属于 Web/后台管理端。

4. 数据导入导出当前更像 Web/教师/管理员端能力。
   - Android 学生端没有直接导入导出入口。
   - 部分 teacher export 接口可用，但 roster import preview、admin export-template、admin deliveries 等接口返回 404。
   - 需要确认 Android 端是否需要承担该模块。

5. 拍照上传在模拟器中受环境限制。
   - 选择照片/视频上传已跑通。
   - 拍照按钮已修复代码问题，并在模拟器无相机时展示提示。
   - 真机拍照仍建议后续使用实体 Android 手机补测。

6. 部分后端字段和前端显示存在不一致。
   - 通知、记录中存在 `course` 等英文枚举值直接出现在用户可见文本中的情况。
   - 部分数值字段以字符串形式返回，例如学时字段。
   - Android 已做部分兼容，但建议后端统一字段规范。

7. 项目源码中仍存在部分中文乱码文案。
   - 当前主要功能可用，但部分页面或提示语仍可能出现乱码。
   - 后续需要统一修复源码编码/文案。
